Michael Weber

Throughout my whole life I've been fascinated with making things. As a kid I used to craft inventions out of broken pens and rubber bands, and later on made my own musical instruments by circuit-bending toys. As an adult I've spent the last ten years working as a professional musician, writing/recording music for TV shows and teaching private and group music lessons.

Once I discovered how challenging and rewarding web development is, though, I never looked back. The feeling of creating something out of thin air, something that other people can interact with and find value in, is what drives me to get up every morning and web development offers me that opportunity like nothing else.

My goals for the future are to continue learning new languages and frameworks, to strengthen my understanding of core HTML, CSS and JavaScript, and find a full-time position that will help me grow as a team-oriented programmer.

5 Day Weather Forecast

View Site View Source

Description: A weather app that takes your zip code and gives you a five day forecast for your location. Made with React and the Open Weather Map API

Technologies Used: HTML, CSS Flexbox, React.js, JavaScript, Axios

Challenges Overcome: With this project I used Axios to connect with an outside web API and, based on the entered location, store weather data in the app's state to display for the user. Not only does the app display the temperature for each day, but it also displays the humidity level and an icon to show wether it is sunny, cloudy, or raining. I also set the background image to change based on the current weather, and I made a custom CSS spinning animation that displays when the app first loads.

What's Next? Based on how the Open Weather API works, searching for location by zip code is not very accurate; it seems to only recognize larger cities. The location is much more accurate if you enter your city code but, since looking up city codes is a pain, I would like to figure out a way to turn the entered zip code into a city code within the app before submitting it to the API. I would also like to expand the functionality so that the user can click on a day and get more detailed weather information.

Michael Weber Music

View Site View Source

Description: An online collection of some of the music compositions I've produced. Users can preview the whole collection on the index page or view more details (including a large visual waveform) on each song's show page, and the music creator can login to add, edit, and delete music.

Technologies Used: HTML, CSS, Grid, Flexbox, JavaScript, Node.js, Express.js, EJS, Wavesurfer.js, MongoDB, Passport.js

Challenges Overcome: This is certainly my biggest project yet! On the front end it utelizes CSS Grid for the basic page layout, Flexbox for arranging all of the songs on the index page, and basic JavaScript for handling the logic behind the audio previews. On each song's show page, I use Wavesurfer.js to display the visual waveform and handle the audio controls.
On the back end, Node.js and Express.js handle all of the CRUD routing while MongoDB stores all of the data (with the exception of the actual music files, which are on an Amazon S3 server). Basic local authentication is handled by Passport.js.

What's Next? The most pertinent issue I'm facing with the app is getting audio to play on each song's show page on mobile. The index page works just fine, but when you go to an individual song's page and click the play button there is no sound (even though the cursor moves along the waveform as though it were playing).
Once audio on mobile is fixed, the next major update for the app is to make the songs searchable through their description and keywords, and possibly have the index page show a list of song collections rather then a list of every song.

What's Cookin?

View Site View Source

Description: An interactive wrapper for the Recipe Puppy API. Users can search for either recipe titles or ingredients, and the app returns ten recipes (see "What's Next?" below).

Technologies Used: HTML, CSS, Grid, JavaScript, Node.js, Express.js, EJS

Challenges Overcome: The goal of this project was to call on an API and present the supplied data in a meaningful way. I utilized Express and Request for the back-end routing and EJS to display data from the API dynamically within the HTML. CSS Grid (specifically it's grid-template-columns: minmax feature) allowed the recipes to display responsively and play nice on mobile.

What's Next? One of the biggest things missing from this app is pagination! The Recipe Puppy API is set up in such a way that it automatically returns only ten results for each search query. To see more results, you have to append "p=2", 3, 4, etc. to the endpoint url. I haven't yet figured out how to gracefully code that into the UI, but a solution is in the works!

Fahrenheit to Celsius Converter

View Site View Source

Description: A handy web app that converts degrees Fahrenheit to degrees Celsius, and vice versa.

Technologies Used: HTML, JavaScript, jQuery, Bootstrap

Challenges Overcome: My goal with this project was to become more familiar with Bootstrap for CSS styling and jQuery for DOM manipulation. Once the logic for converting the temperature was figured out, I used jQuery's text(), hasClass() and toggleClass() methods to display the converted data and switch states between "Fahrenheit-to-Celsius" and "Celsius-to-Fahrenheit".

What's Next? I wanted to utelize Bootstrap as much as possible for this app (in fact, I didn't even include my own stylesheet), but I know there is a lot more room to be creative with the design and styling. My next step for this project is to customize the look and feel of it. There is especially a lot of room to draw attention to the switch between conversion states, since I consider this to be the most interesting feature of the app.

Personal Developer Portfolio

View Site View Source

Description: The site you're on right now! A simple, single-page static website to showcase myself as a developer.

Technologies Used: HTML, CSS, Flexbox, CSS Grid, JavaScript

Challenges Overcome: Apart from the design, the trickiest part about making this site was coding the pop-up modal. In order to keep the code light and scalable, I chose to use the getElementsByClassName selector to return node lists of all of the buttons and modals. Then I ran through them all with a for-loop and put the event listeners inside. This way I don't have to write a brand new block of JavaScript for each future project I add; once I add it to the HTML, the pop-up modal will automatically work.

What's Next? Although I fear I'll never be satisfied with the design and the font-choices, my biggest issues so far with the site are compatability between different browsers. One such example is smooth-scrolling, which only works on Chrome right now. Even though I like the simplicity of handling the scrolling behavior with just pure CSS, there are probably third-party JavaScript plugins that to help me out with this.
